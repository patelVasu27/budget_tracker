    ```
    WORD_MAP: Record<string, number> = map of one→1 ... ninety→90
    SCALE_MAP: Record<string, number> = hundred→100, thousand→1000

    function parseAmount(text: string): number | null
      1. Lowercase, trim.
      2. Pass 1 (digit regex): /\d+(?:\.\d{1,2})?/g
         - If matches found, filter > 0, return Math.max(...amounts)
      3. Pass 2 (word-to-number):
         - Strip non-alpha whitespace, split by spaces
         - Accumulate: for each word, if WORD_MAP → add to current; if SCALE_MAP → multiply; else flush current to total
         - Return total if > 0, else null
    ```

    ```
    CATEGORY_KEYWORDS: Record<string, string[]> = map from CONTEXT.md
    - Food: food, groceries, lunch, dinner, breakfast, eat, restaurant, cafe
    - Transport: transport, gas, fuel, uber, taxi, bus, train, parking
    - Utilities: utilities, electricity, water, bill, phone, internet
    - Entertainment: entertainment, movies, games, netflix, spotify, concert
    - Shopping: shopping, clothes, amazon, store, mall
    - Health: health, doctor, pharmacy, medicine, gym

    function matchCategory(transcript: string): { bestGuess: string; scores: Record<string, number> }
      1. Initialize all 7 categories with score 0.
      2. Lowercase transcript.
      3. For each category+keywords, check if keyword appears in transcript (string.includes). If yes, increment score.
      4. Sort by score descending, name ascending (tiebreaker). Return { bestGuess: highest, scores: allScores }
    ```

| T-03-01 | Information Disclosure | Web Speech API transcription | accept | Raw audio is processed by the browser's built-in speech service (Google servers for Chrome). The application never stores or transmits the raw audio. The transcript exists only in JS memory until the user confirms the modal. This is standard Web Speech API behavior documented by MDN. |
| T-03-02 | Tampering | Transcript parsing (amount + category) | mitigate | Parsed voice data follows the same validation path as manual entry: react-hook-form validates on the `amount` field (`required`, `min: '0.01'`), and the Supabase insert is subject to RLS (AUTH-05). Parsers operate on string values only, never executing untrusted input. |
| T-03-03 | Information Disclosure | Transcript displayed in note field | mitigate | The raw transcript is pre-filled into the modal's `note` field. The user reviews and can edit before saving. The note field accepts arbitrary text (same as manual entry). No new XSS surface — React's JSX auto-escapes. |
| T-03-04 | Elevation of Privilege | Mic button state management | mitigate | The `useVoiceCommand` hook only starts/stops the browser's SpeechRecognition API. It never interacts with Supabase or performs mutations. All database writes go through the existing `addMutation` path with RLS enforcement. |
| T-03-05 | Denial of Service | Microphone permission denied | accept | Browser caches mic permission. If denied, the mic button shows a disabled state with a tooltip explaining how to re-enable. This is a UX degradation, not a security vulnerability. |
</threat_model>
